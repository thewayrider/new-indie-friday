# Project Brief: Domain Consolidation & Migration

**Project Focus:** Integrating `streamusique.com` into `kimrampling.com`

## 1. Executive Summary

The objective of this strategy is to transition a low-volume, Blogger-hosted music blog (`streamusique.com`) into a centralized personal brand and music discovery platform hosted under a personal domain (`kimrampling.com`).

By implementing a strategic redirect, the project will establish **Kim Rampling** as the primary SEO destination while preserving the inherent branding and semantic value of the "Streamusique" name, which aligns perfectly with modern music consumption habits.

---

## 2. Technical Feasibility & Verdict

* **Status:** Fully Feasible / Recommended.
* **Complexity:** Low to Moderate (Can be fully configured in less than an hour once the new site is live).
* **Core Philosophy:** Rather than letting the old domain expire, it will be repurposed as a high-value marketing pipeline that feeds traffic and authority directly into the new personal brand.

---

## 3. Implementation Blueprint

### A. The Redirect Architecture

To achieve the best user experience and protect search engine rankings, the migration will utilize **301 (Permanent) Redirects**.

```
[ Visitor / Search Engine ] 
           │
           ▼
   streamusique.com (Old Domain)
           │
           ▼  (301 Permanent Redirect via DNS)
   kimrampling.com/music (Dedicated Deep-Link Destination)

```

### B. Execution Steps

1. **Registrar-Level Forwarding:** Log into the domain registrar for `streamusique.com`, disconnect it from the Blogger servers, and set up a permanent 301 domain-forwarding rule to the new destination.
2. **Blogger Decommissioning (Optional Fallback):** Revert the Blogger site to a standard `.blogspot.com` subdomain. If legacy blog posts still receive traffic, implement a JavaScript redirect script in the Blogger HTML header to catch and forward remaining visitors.

---

## 4. Strategic & SEO Best Practices

* **Avoid the "Catch-All" Homepage Trap:** To prevent search engines from ignoring the redirect (treating it as a "Soft 404") and to keep users from getting confused, traffic should ideally **not** be sent to the generic root homepage (`kimrampling.com`).
* **The "Contextual Landing Page" Strategy:** Traffic from `streamusique.com` will be mapped directly to a highly relevant, dedicated section of the new site (e.g., `[kimrampling.com/music](https://kimrampling.com/music)` or `/streamusique`).
* **The Benefit:** This maintains keyword and semantic relevance. Google will understand exactly why a domain about "streaming music" is pointing to that specific page, seamlessly transferring old authority to the new website.
